Local Maestro automation, with a dashboard worth opening

maestRoRun wraps Maestro with Node.js orchestration, saved tests and suites, local reports, optional Slack summaries, and a React dashboard that helps you run mobile flows without rebuilding your own tooling every sprint.

View on GitHub Get Started

Features

Everything you need to run Maestro locally and keep the team in the loop.

Dashboard-first workflow

Trigger runs, manage saved tests/suites, inspect reports, and edit local settings from one UI.

Android & iOS ready

Run against Android Emulator and iOS Simulator with device discovery built into the dashboard.

Reports & screenshots

Store JSON summaries, JUnit output, and Maestro debug screenshots under `reports/`.

Slack summaries

Send run summaries through Incoming Webhooks when a test batch finishes.

Nightly scheduling

Save, install, or remove a local cron entry directly from the dashboard.

Starter-safe repo

No private flows, credentials, or proprietary library data ship with the public repo.

Architecture

A small stack with a clear split between UI, orchestration, and Maestro execution.

1. Dashboard React + Vite UI for config, library, reports, and run controls.
2. API Express server orchestrates devices, env, reports, and cron.
3. Maestro Executes YAML flows on Android Emulator or iOS Simulator.

Quick Start

Clone the repo, install dependencies, copy the example env file, and open the dashboard.

npm install cp .env.example .env npm run dashboard

Then open http://127.0.0.1:5173.
